# ProjectExpert - AI项目专家部门

> 一个覆盖全行业的AI智能体团队，通过对话引导式交互，帮助用户快速了解、分析和规划任何领域的项目。

## 🎯 团队定位

AI项目专家部门是一个综合性的项目分析智能体团队，能够：

- 🔍 **轮询识别项目意图**：通过逐个问题的对话，精准识别用户想了解的项目
- 📍 **精准定位项目**：锁定用户真正要深入了解的具体项目
- 🌐 **全面检索信息**：跨行业、多维度地搜集项目相关信息
- 🏗️ **输出技术栈与知识库**：梳理项目技术体系，制定学习推进计划
- 📐 **需求分析与规划**：系统化的需求分析和可执行的项目推进方案
- 💡 **关键信息降智**：用大白话让任何人都能快速理解新项目

## 🚀 使用方式

### 方式一：通过 Agent 入口（推荐）

1. 在 Trae 中打开本项目文件夹
2. 在对话中输入 `@ProjectExpert` 调用智能体
3. 直接描述你想了解的项目（越模糊越好，智能体会引导你）
4. 智能体会自动按流程完成所有分析和输出

### 方式二：通过 Rules 入口

1. 在 Trae 中打开本项目文件夹
2. Trae 会自动加载 `.trae/rules/` 下的所有规则文件
3. `director-main.md` 作为主控规则，包含完整的工作流程
4. 直接描述你想了解的项目即可启动

### 工作流程

```
用户输入 → 需求引导 → 情报检索 → 规划分析 → 降智输出 → 最终交付
```

1. **需求引导**：意图探针逐个问题轮询 → 项目定位师锁定目标
2. **情报检索**：全域侦察兵多维检索 → 知识图谱师结构化整理
3. **规划分析**：技术架构师梳理技术栈 → 需求规划师制定方案
4. **降智输出**：降智翻译官输出通俗总结
5. **最终交付**：整合所有成果，生成完整报告

## 🔧 架构说明

### 为什么这样设计？

Trae 的 Rules 体系会自动加载 `.trae/rules/` 下所有 `.md` 文件作为 AI 的上下文，但**不会自动按流程"调用"某个规则**。因此：

- **`director-main.md`** 作为主控规则，**内联了完整的四阶段工作流程**，确保 AI 无论是否能访问子规则文件，都能按完整流程执行
- **`.trae/agents/project-expert.yaml`** 作为 Agent 入口，通过 `@ProjectExpert` 一键调用，绑定所有规则文件
- **子目录下的规则文件**（`guidance/`、`intelligence/` 等）作为各角色的详细行为规范，AI 在扮演对应角色时参考使用
