# 项目发现方法论

> 本文档定义了AI项目专家部门在项目发现阶段的标准方法论。

## 一、核心原则

### 1.1 漏斗式收敛原则
项目发现是一个从宽泛到具体的收敛过程，每一层都缩小范围，直到精确定位目标项目。

```
行业领域 → 项目类型 → 具体方向 → 项目定位
  (宽)                                      (窄)
```

### 1.2 一次一问原则
每次只问一个问题，避免用户认知过载。问题之间有明确的逻辑递进关系。

### 1.3 意图 vs 表达分离原则
用户表达的需求 ≠ 真实意图。必须通过多轮对话挖掘深层意图。

## 二、项目发现流程

### 阶段1：行业识别（1-2轮对话）

**目标**：确定项目所属行业领域

**提问策略**：
- 直接问："你想了解的项目属于哪个行业？"
- 提供选项辅助：软件/互联网、游戏、硬件/电子、建筑/结构、金融、制造、医疗、教育、其他

**判断标准**：
- ✅ 用户明确说出行业 → 进入阶段2
- ❌ 用户说不清楚 → 用具体例子引导："比如你想做一个App，还是想了解芯片设计？"

### 阶段2：类型识别（1-2轮对话）

**目标**：确定项目类型

**提问策略**：
- 根据行业动态调整问题
- 软件行业："你想做Web应用、移动App、桌面软件还是后端服务？"
- 游戏行业："你对手游、端游、主机游戏还是独立游戏更感兴趣？"
- 硬件行业："你关注消费电子、工业设备、嵌入式系统还是芯片设计？"

**判断标准**：
- ✅ 用户明确说出类型 → 进入阶段3
- ❌ 用户不确定 → 提供对比说明帮助选择

### 阶段3：方向聚焦（1-2轮对话）

**目标**：缩小到具体的项目方向

**提问策略**：
- "能再具体一点吗？比如在{类型}中，你最关注{方向A}还是{方向B}？"
- 如果用户已经描述得很具体，可以直接跳到确认

**判断标准**：
- ✅ 能唯一定位一个项目方向 → 进入阶段4
- ❌ 仍有多个可能 → 列出候选让用户选择

### 阶段4：目的确认（1轮对话）

**目标**：了解用户了解这个项目的目的

**提问策略**：
- "你了解这个项目的目的是什么？"
- 选项：学习入门、评估可行性、准备启动项目、技术选型、其他

**判断标准**：
- 目的直接影响后续分析的深度和侧重点

### 阶段5：水平评估（1轮对话）

**目标**：了解用户当前的知识水平

**提问策略**：
- "你目前对这个领域了解多少？"
- 选项：完全零基础、有一些基础概念、有一定实践经验、比较熟悉

**判断标准**：
- 水平直接影响学习计划的起点和技术解释的深度

### 阶段6：关注确认（1轮对话）

**目标**：了解用户最关注的方面

**提问策略**：
- "在项目分析中，你最希望重点关注哪些方面？"
- 选项：技术栈、学习路径、市场需求、实施难度、成本估算、全部都要

## 三、项目定位方法

### 3.1 特征匹配法
1. 从对话中提取关键特征词
2. 在项目知识库中搜索匹配项
3. 计算匹配度分数
4. 按分数排序推荐

### 3.2 置信度评估
| 置信度 | 条件 | 处理方式 |
|--------|------|----------|
| 高（≥80%） | 特征高度匹配，用户描述清晰 | 直接推荐 |
| 中（60-80%） | 特征部分匹配，有多个可能 | 列出候选让用户选择 |
| 低（<60%） | 信息不足或过于模糊 | 继续提问补充信息 |

### 3.3 跨行业识别
当项目可能跨越多个行业时：
- 识别项目的主要行业属性
- 标注次要相关行业
- 在分析中综合考虑多行业因素

---

## 四、常见问题处理

### Q1：用户描述过于模糊
**处理**：用具体例子引导，提供选项辅助选择

### Q2：用户同时想了解多个项目
**处理**：建议逐个分析，先选择最想了解的一个

### Q3：用户描述的需求不存在对应项目
**处理**：分析用户真实意图，推荐最接近的项目方向

### Q4：用户中途改变想法
**处理**：灵活调整，基于已有信息重新定位
