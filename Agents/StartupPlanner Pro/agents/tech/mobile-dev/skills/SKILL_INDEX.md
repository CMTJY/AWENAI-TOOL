# tech-mobile-dev 本地技能索引

本目录只服务 `tech-mobile-dev`。收到任务后先读取本索引，再完整读取与当前任务匹配的技能。禁止读取 StartupPlanner Pro 外部的 Superpowers 源包。

| Skill | 用途 |
|---|---|
| [ui-ux-pro-max](./ui-ux-pro-max/SKILL.md) | 系统检查可用性、可访问性、响应式、平台和界面状态。 |
| [test-driven-development](./test-driven-development/SKILL.md) | 用可观察的失败测试驱动行为实现和缺陷修复。 |
| [systematic-debugging](./systematic-debugging/SKILL.md) | 先用证据和单一假设定位根因，再实施修复。 |
| [receiving-code-review](./receiving-code-review/SKILL.md) | 核验、处理并记录独立审查反馈。 |
| [verification-before-completion](./verification-before-completion/SKILL.md) | 交接和完成结论前运行新鲜验证。 |

## 选择规则

- 一般实现加载 TDD 与完成验证；仅在真实失败时加载系统调试。
- 只有收到独立审查反馈时加载 receiving-code-review。
- UI 技能只在界面任务使用；frontend-design 负责艺术方向，ui-ux-pro-max 负责系统体验质量。
- 角色文件、Task Packet、文件范围和验收标准始终约束技能。
- 本地副本独立维护，不会随源包自动更新。
