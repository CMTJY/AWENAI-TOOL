# 跨行业检索方法论

> 本文档定义了AI项目专家部门在情报检索阶段的标准方法论。

## 一、核心原则

### 1.1 多维交叉原则
从技术、市场、社区、学术、实践、资源六个维度交叉检索，确保信息覆盖无死角。

### 1.2 质量优先原则
优先获取A级（官方文档、学术论文）和B级（知名技术博客）信息源。

### 1.3 时效性原则
技术信息优先近2年，市场信息优先近1年，基础概念不限时效。

---

## 二、六维检索框架

### 维度1：技术维度
**检索目标**：技术栈、架构、工具链、开发框架

**信息源优先级**：
1. 官方文档和API文档
2. GitHub仓库（按Star数排序）
3. 技术博客（Medium、Dev.to、掘金、InfoQ）
4. Stack Overflow 热门问题

**检索关键词模板**：
- "{项目名} 技术栈"
- "{项目名} architecture"
- "{项目名} tech stack {年份}"
- "{技术名} best practices"

### 维度2：市场维度
**检索目标**：市场规模、竞品、趋势、用户需求

**信息源优先级**：
1. 行业研究报告（Gartner、IDC、艾瑞等）
2. 新闻媒体（TechCrunch、36氪等）
3. 投资机构报告
4. 产品评测和对比文章

**检索关键词模板**：
- "{项目/行业} market size {年份}"
- "{项目/行业} trends {年份}"
- "{项目名} competitors"
- "{项目/行业} 投资报告"

### 维度3：社区维度
**检索目标**：社区活跃度、开发者评价、常见问题

**信息源优先级**：
1. Reddit（相关subreddit）
2. Hacker News
3. 知乎、V2EX、SegmentFault
4. Discord/Slack 社区

**检索关键词模板**：
- "{项目名} reddit"
- "{项目名} review"
- "{项目名} 评价"
- "{项目名} 常见问题"

### 维度4：学术维度
**检索目标**：相关论文、研究成果、前沿技术

**信息源优先级**：
1. Google Scholar
2. arXiv（计算机科学）
3. IEEE Xplore
4. ACM Digital Library

**检索关键词模板**：
- "{项目/技术名} survey"
- "{项目/技术名} research paper"
- "{项目/技术名} state of the art"

### 维度5：实践维度
**检索目标**：案例研究、最佳实践、踩坑经验

**信息源优先级**：
1. 技术博客的实战系列
2. 开源项目的README和Wiki
3. 会议演讲（YouTube、SlideShare）
4. 公司技术博客（Uber Engineering、Netflix Tech Blog等）

**检索关键词模板**：
- "{项目名} case study"
- "{项目名} lessons learned"
- "{项目名} best practices"
- "how to {项目相关动作}"

### 维度6：资源维度
**检索目标**：学习资源、教程、课程、书籍

**信息源优先级**：
1. 官方教程和文档
2. 在线课程平台（Coursera、Udemy、慕课网等）
3. 书籍推荐（Goodreads、豆瓣）
4. GitHub Awesome列表

**检索关键词模板**：
- "{项目/技术名} tutorial"
- "{项目/技术名} 学习路线"
- "best {项目/技术名} books"
- "awesome-{项目/技术名}"

## 三、行业适配策略

### 软件行业
- **重点维度**：技术、社区、实践
- **特色信息源**：GitHub、Stack Overflow、技术博客
- **检索特点**：信息丰富，注重技术细节和社区反馈

### 游戏行业
- **重点维度**：技术、市场、资源
- **特色信息源**：GDC Vault、游戏开发社区、Steam数据
- **检索特点**：关注引擎选型、美术管线、市场数据

### 硬件行业
- **重点维度**：技术、学术、实践
- **特色信息源**：芯片厂商文档、专利数据库、EDA工具文档
- **检索特点**：关注芯片方案、供应链、认证标准

### 建筑/结构行业
- **重点维度**：技术、学术、资源
- **特色信息源**：规范标准库、BIM资源、学术论文
- **检索特点**：关注规范标准、材料技术、BIM

### 金融行业
- **重点维度**：市场、学术、实践
- **特色信息源**：监管文件、金融科技报告、安全标准
- **检索特点**：关注合规要求、安全标准、监管动态

## 四、信息质量评估

### 可信度等级
| 等级 | 定义 | 典型来源 |
|------|------|----------|
| A级 | 权威、经过同行评审 | 官方文档、学术论文、权威报告 |
| B级 | 专业、有一定影响力 | 知名技术博客、行业专家文章 |
| C级 | 个人观点、未经验证 | 社区讨论、个人博客 |
| D级 | 信息不明、需谨慎对待 | 匿名来源、未经验证的声称 |

### 时效性评估
| 时效 | 适用信息类型 |
|------|-------------|
| 近6个月 | 市场数据、新闻、版本更新 |
| 近1年 | 技术趋势、社区动态 |
| 近2年 | 技术实践、教程、工具 |
| 2年以上 | 基础概念、经典理论、设计原则 |

## 五、检索预算控制

| 项目类型 | 总检索次数 | 每维度平均 |
|----------|-----------|-----------|
| 简单项目 | 15-20次 | 2-3次 |
| 中等项目 | 25-35次 | 4-6次 |
| 复杂项目 | 40-50次 | 6-8次 |

**优化策略**：
- 先用宽泛关键词检索，再逐步精确
- 优先检索高质量信息源
- 发现高价值信息后深入追踪引用
