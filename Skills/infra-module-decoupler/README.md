# infra-module-decoupler

基础设施模块解耦器 — 一个 Trae AI 技能，用于分析和清理嵌入式 C 项目中基础设施模块的违规依赖。

## 功能

- 分析任意目标模块的完整依赖关系（被谁依赖、依赖谁）
- 识别对非系统文件的违规依赖
- 提供 3 种标准化解耦策略（死代码删除、系统API替换、业务逻辑归还）
- 生成完整的修改方案报告（包含依赖关系图、影响范围、死代码清单）
- 逐步执行代码修改，每次修改后自动验证
- 最终全局验证模块独立性

## 适用场景

- ESP-IDF / 嵌入式 C 项目
- 需要提取日志模块、存储模块、缓存模块等作为独立可复用的基础设施
- 重构遗留代码，清理跨层依赖
- 项目模块化拆分前的依赖分析

## 使用方法

在 Trae IDE 中，直接对话即可触发：

```
"分析 logger.c、flash_cache.c、nvs_storage.c 三个模块的依赖关系"
"把日志模块的违规依赖全部清理干净"
"让 nvs_storage 模块只能被依赖，不能依赖其他业务模块"
```

## 工作流程

```
Phase 1: 依赖分析  →  读取模块，全项目搜索引用关系
Phase 2: 违规识别  →  区分系统依赖 vs 业务依赖
Phase 3: 方案设计  →  匹配策略，生成报告 → 用户确认
Phase 4: 逐步执行  →  逐点修改 + 验证 → 全局验证
```

## 真实案例

此技能源自对一个 ESP-IDF 智能花盆项目的依赖重构，成功解耦了 3 个模块：

| 模块 | 违规数 | 处理方式 | 结果 |
|------|:----:|---------|------|
| logger | 1 | 死代码删除 | -15行 |
| flash_cache | 1 | 系统API替换 | 宏替换25处 |
| nvs_storage | 3 | 系统API替换+业务归还 | -535行 |

最终 3 个模块均实现：仅依赖 ESP-IDF 系统文件 + C标准库，可独立编译提取。
