# 需求意图识别文档 - 输出模板

本文档是 `intent-recognition` 技能输出的标准格式参考。

---

## 标准输出格式

```markdown
# 需求意图识别文档

## 原始需求描述

> "[用户最初的需求描述原文]"

---

## 需求意图分析

### 一、产品/项目信息

| 属性 | 详情 |
|------|------|
| 产品名称 | [名称] |
| 产品形态 | [实体/数字/服务/平台/其他] |
| 目标人群 | [目标用户画像] |
| 当前阶段 | [想法/原型/完成/已上线] |

### 二、核心目标

| 优先级 | 目标 | 说明 |
|--------|------|------|
| A | [目标名称] | [具体说明] |
| B | [目标名称] | [具体说明] |
| C | [目标名称] | [具体说明] |

### 三、商业逻辑/用户场景

[描述用户的使用场景、商业背景或业务流程]

---

## 深层需求解读

### 表面需求
[用户明确表达出来的需求]

### 真正意图
[用户真正想要达成的深层目标，往往隐藏在表面需求之下]

---

## 注意事项

1. [任何需要提醒用户的重要事项]
2. [例如：本技能仅识别意图，不提供解决方案]
```

---

优先保存到以下路径：

2. 当前工作目录下的 `需求意图识别.md`

## 使用示例

### 示例 1：潮玩产品

```markdown
# 需求意图识别文档

## 原始需求描述

> "我现在已经设计好了一个产品，但是我想策划一些活动把我这个产品曝光出去进行营销好，后面方便售卖"

---

## 需求意图分析

### 一、产品/项目信息

| 属性 | 详情 |
|------|------|
| 产品名称 | 潮玩盆栽 |
| 产品形态 | 实体产品 |
| 目标人群 | 年轻白领、学生、潮流爱好者、绿植爱好者 |
| 当前阶段 | 已完成设计 |

### 二、核心目标

| 优先级 | 目标 | 说明 |
|--------|------|------|
| A | 获取曝光 | 让更多人知道这个产品/品牌 |
| B | 收集用户 | 积累潜在客户名单 |
| C | 直接卖货 | 产生销量（AB做好后自然实现） |

### 三、商业漏斗逻辑

```
曝光(A) → 吸引兴趣(B) → 转化销售(C)
```

---

## 深层需求解读

### 表面需求
策划营销活动来曝光产品，便于后续售卖

### 真正意图
- 打造品牌知名度 + 积累第一批种子用户
- 处于 0→1 冷启动阶段，需要从零打开市场认知
- 核心是市场教育和品牌认知的建立
```

### 示例 2：SaaS 产品

```markdown
# 需求意图识别文档

## 原始需求描述

> "我们需要一个客户管理系统"

---

## 需求意图分析

### 一、产品/项目信息

| 属性 | 详情 |
|------|------|
| 产品名称 | 客户管理系统 |
| 产品形态 | SaaS 软件 |
| 目标人群 | 销售团队、运营人员 |
| 当前阶段 | 想法阶段 |

### 二、核心目标

| 优先级 | 目标 | 说明 |
|--------|------|------|
| A | 提升效率 | 减少手动录入时间 |
| B | 数据可视化 | 更好地了解客户情况 |
| C | 团队协作 | 销售和运营信息同步 |

---

## 深层需求解读

### 表面需求
需要一个客户管理系统

### 真正意图
解决销售团队信息分散、难以跟踪客户进度的问题，提升团队协作效率
```

---

## 关键原则

1. **保留用户原话** - 原始需求描述使用用户的原文引用
2. **区分表面与深层** - 明确区分用户"说的"和"真正想要的"
3. **不提供方案** - 文档只识别意图，不包含解决方案或实施建议
4. **简洁清晰** - 使用表格和列表，保持文档易读
