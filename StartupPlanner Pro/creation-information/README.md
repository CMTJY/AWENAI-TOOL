## 项目简介

**StartupPlanner** 是一个专为创业者设计的智能体协作平台，通过模拟企业六大核心部门的协作流程，帮助创业者系统性地完成从0到1的创业规划。

无论你是处于想法期、验证期还是启动期，StartupPlanner 都能为你提供：

- 系统化的创业规划方法论
- 12+ 专业智能体的协同服务
- 标准化的商业计划书输出
- 可落地的执行路线图

## 核心特色

### 六大部门协同
平台模拟真实企业的六大核心部门，每个部门由专业的AI智能体负责：

| 部门 | 核心职能 | 智能体数量 |
|------|----------|------------|
| 战略部 | 行业分析、商业模式设计 | 2 |
| 产品部 | 产品规划、MVP设计 | 2 |
| 营销部 | 市场推广、品牌建设 | 1+ |
| 财务部 | 财务建模、融资规划 | 2 |
| 组织部 | 团队架构、股权设计 | 2 |
| 风险部 | 风险识别、退出机制 | 1 |

### 标准化交付物

每个阶段都有明确的标准化输出模板：

- **行业分析报告** - 包含PEST分析、市场规模测算、竞争格局
- **用户画像文档** - 覆盖人口统计、行为特征、心理特征
- **商业模式画布** - 完整的9要素商业模型
- **产品需求文档(PRD)** - 功能规划、用户故事、路线图
- **财务预测模型** - 3年滚动预测、盈亏平衡分析
- **商业计划书** - 完整的11章节BP文档

## 架构说明

### 六大部门详解

#### 1. 战略部 (Strategy)
负责创业方向的顶层设计和商业逻辑验证。

**核心智能体：**
- **战略分析师** - 行业洞察与商业模式设计专家，擅长赛道评估、竞争格局分析
- **市场调研员** - 用户需求洞察与市场规模测算专家，擅长用户画像构建和需求验证

**主要输出：**
- 行业分析报告
- 商业模式画布
- 市场进入策略

#### 2. 产品部 (Product)
负责产品规划、功能设计和MVP定义。

**核心智能体：**
- **产品经理** - 负责产品定位、功能规划、用户体验设计和产品路线图制定
- **MVP设计师** - 专注于最小可行产品设计和快速验证方案

**主要输出：**
- 产品需求文档(PRD)
- 产品路线图
- MVP功能清单

#### 3. 营销部 (Marketing)
负责市场推广策略和品牌建设。

**核心智能体：**
- **营销策划师** - 负责整体营销策略制定
- **增长黑客** (规划中) - 产品增长与活动策划
- **内容创作者** (规划中) - 媒体内容运营
- **私域运营师** (规划中) - 私域流量运营

**主要输出：**
- 营销策略方案
- 获客渠道规划
- 品牌建设方案

#### 4. 财务部 (Finance)
负责财务建模、成本控制和融资规划。

**核心智能体：**
- **财务分析师** - 专业的创业财务规划专家，负责成本结构分析、收入模型设计
- **融资顾问** - 专注于融资策略和投资人沟通

**主要输出：**
- 财务预测模型
- 成本结构分析
- 融资方案

#### 5. 组织部 (Organization)
负责团队架构设计和股权分配。

**核心智能体：**
- **组织架构师** - 专注于创业团队架构设计、岗位规划、股权分配
- **人才规划师** - 负责人才招聘策略和团队建设

**主要输出：**
- 组织架构图
- 股权分配方案
- 岗位说明书

#### 6. 风险部 (Risk)
负责风险识别、评估和应对预案。

**核心智能体：**
- **风险管理师** - 负责创业项目全生命周期风险识别、评估、应对与退出机制设计

**主要输出：**
- 风险识别清单
- 风险评估矩阵
- 应对预案

### 总指挥体系

**创业总指挥**是整个规划流程的协调者，负责：

- 项目诊断与启动
- 阶段管理与质量把控
- 部门协调与冲突仲裁
- 最终BP整合输出

## 快速开始指南

### 第一步：启动项目诊断

与**创业总指挥**对话，完成项目基本信息收集：

```
1. 项目定位：核心功能是什么？
2. 当前阶段：想法期/验证期/启动期？
3. 目标用户：主要面向哪类人群？
4. 资源情况：团队规模？资金预算？
5. 规划目标：用于融资/团队招募/自我梳理？
```

### 第二步：按阶段推进

根据总指挥制定的路线图，按三阶段逐步推进：

**第一阶段：方向探索**
- 调用战略部完成行业分析和商业模式设计
- 输出：行业分析报告 + 商业模式画布

**第二阶段：方案设计**
- 调用产品部完成产品规划
- 调用营销部制定营销策略
- 调用组织部设计团队架构
- 调用财务部建立财务模型

**第三阶段：整合输出**
- 调用风险部完成风险分析
- 调用财务部完成融资方案
- 总指挥汇总输出完整BP

### 第三步：审核与迭代

每阶段完成后进行质量门禁检查，不通过则回溯调整。

## 使用示例

### 示例场景：宠物社交平台创业规划

**用户输入：**
> "我想做一个宠物社交平台"

**创业总指挥响应：**

```markdown
好的，我来帮你系统规划这个项目。首先我需要了解一些基本信息：

1. **项目定位**：这个宠物社交平台的核心功能是什么？
   （例如：宠物社交/宠物医疗/宠物电商/综合平台）

2. **当前阶段**：
   - 想法期：只有初步想法
   - 验证期：已做过一些用户调研
   - 启动期：已有产品原型或初步用户

3. **目标用户**：
   - 主要面向哪类宠物主？（猫/狗/异宠）
   - 用户年龄段？（Z世代/白领/中老年）
   - 地域范围？（本地/全国/出海）

4. **资源情况**：
   - 团队规模？
   - 资金预算？
   - 期望多久完成商业计划书？

5. **规划目标**：
   - 本次规划主要用于？（融资/团队招募/自我梳理）
   - 期望融资轮次？

请回答以上问题，我将为你制定详细的规划路线图。
```

**后续流程：**

```
用户回答 → 生成 project-brief.md → 第一阶段启动
                                              ↓
【调用 战略部-市场调研员】用户画像与市场规模
         ↓
【调用 战略部-战略分析师】行业分析与商业模式
         ↓
质量门禁检查 → 进入第二阶段
         ↓
【调用 产品部-产品经理】产品规划
【调用 产品部-MVP设计师】MVP设计
【调用 财务部-财务分析师】财务模型
         ↓
质量门禁检查 → 进入第三阶段
         ↓
【调用 风险部-风险管理师】风险分析
【调用 财务部-融资顾问】融资方案
         ↓
总指挥整合输出完整BP
```

## 智能体使用规范

### 调用格式

当需要调用某部门时，使用以下格式：

```
【调用 {部门名}】
输入：
- 前序输出：{引用之前部门的输出}
- 当前需求：{明确本次调用的任务}
- 约束条件：{预算、时间、资源等约束}

期望输出：{明确期望的交付物}
```

### 质量门禁检查清单

**第一阶段门禁：**
- [ ] 市场规模数据有来源
- [ ] 用户画像有调研支撑
- [ ] 商业模式画布完整
- [ ] 盈利模式清晰可行

**第二阶段门禁：**
- [ ] 产品功能与用户需求匹配
- [ ] 营销策略有具体数字目标
- [ ] 组织架构与阶段匹配
- [ ] 财务模型3年预测完整
- [ ] 盈亏平衡点可计算

**第三阶段门禁：**
- [ ] BP覆盖全部11个章节
- [ ] 各章节数据一致
- [ ] 风险矩阵完整
- [ ] 执行路线图有时间节点

## 适用人群

- **早期创业者** - 处于想法期或验证期，需要系统化梳理商业逻辑
- **融资准备者** - 准备天使轮或Pre-A轮融资，需要专业BP
- **团队组建者** - 正在组建核心团队，需要股权和架构设计
- **项目转型者** - 现有项目需要重新定位或调整方向

## 最佳实践

1. **完整经历三阶段** - 不要跳过任何阶段，每个阶段都是后续的基础
2. **重视质量门禁** - 不通过门禁检查不要进入下一阶段
3. **保持数据一致** - 各部门使用的数据要保持一致，避免冲突
4. **主动提供信息** - 越详细的项目信息越能得到精准的建议
5. **迭代优化** - 根据反馈不断调整和优化方案
