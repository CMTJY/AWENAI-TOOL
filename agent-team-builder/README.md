# agent-team-builder

> 通用智能体团队搭建技能

## 简介

通过"需求识别→框架规划→实际搭建"三阶段流程，帮助用户从零搭建任何领域的 AI 智能体团队。

## 适用场景

- 创业规划智能体团队
- 营销运营智能体团队
- 研发管理智能体团队
- 内容创作智能体团队
- 任何需要多智能体协作的场景

## 核心能力

| 能力 | 说明 |
|------|------|
| 需求意图识别 | 通过引导式提问，明确用户真实需求 |
| 部门架构设计 | 设计平级部门体系，避免职责重叠 |
| 智能体角色定义 | 定义每个智能体的角色、职责、输入输出 |
| 协作流程规划 | 设计部门间调用关系和数据流 |
| 规则文件创建 | 创建专业、可执行的智能体规则文件 |
| 模板生成 | 生成输出模板和方法论文档 |

## 三阶段流程

```
阶段一：需求意图识别
├── 引导式提问（6-8个问题）
├── 明确团队用途、场景、输出
└── 输出：需求意图识别文档
         ↓
阶段二：框架规划
├── 设计部门架构
├── 定义智能体角色
├── 设计协作流程
└── 输出：搭建框架文档（支持迭代修正）
         ↓
阶段三：实际搭建
├── 检索现有资源（GitHub）
├── 创建智能体规则文件
├── 生成模板和方法论文档
└── 输出：完整项目工程
```

## 使用方式

在对话中调用此技能：

```
Use Skill: agent-team-builder
```

然后描述你想搭建的智能体团队，技能会引导你完成整个搭建过程。

## 关键设计原则

1. **一次只问一个问题** — 引导式提问
2. **用户确认再推进** — 每阶段需确认
3. **平级部门架构** — 无从属关系
4. **避免职责重叠** — 及时合并
5. **支持迭代修正** — 框架可多次调整
