# intent-recognition

意图识别技能 - 通过引导式提问帮助用户理清真实需求

## 技能描述

通过"一次一个问题"的引导式提问，逐步挖掘用户的**真正核心意图**，并输出结构化的需求意图识别文档。

## 使用场景

- 用户需求不明确，需要帮助理清真正想要什么
- 业务咨询场景，需要识别客户的真实诉求
- 产品规划前，帮助团队对齐对需求的理解
- 任何需要从模糊描述中提取明确意图的场景

## 核心特性

1. **引导式提问** - 每次只问一个问题，让用户逐一回答
2. **追问逻辑** - 挖掘用户话语背后的真正目的
3. **中立立场** - 不预设立场，让用户自己决定方向
4. **文档化输出** - 生成结构化的需求意图识别文档

## 使用方法

当用户有模糊需求需要理清时，调用此技能：

```bash
Skill: intent-recognition
```

技能会：
1. 欢迎用户并说明作用
2. 通过 4-6 个关键问题逐步澄清需求
3. 总结确认理解是否正确
4. 输出需求意图识别文档

## 输出文档

技能会生成 `需求意图识别.md` 文件，包含：

- 原始需求描述
- 产品/项目信息
- 核心目标分析
- 深层需求解读（表面需求 vs 真正意图）

## 约束

- ✅ 只负责识别意图
- ✅ 输出结构化文档
- ❌ 不提供解决方案
- ❌ 不输出实施建议

## 示例

**输入**：用户说"我想做点营销推广"

**输出**：经过引导提问后，生成需求意图识别文档，明确用户真正想要的是"打造品牌知名度 + 积累第一批种子用户"
